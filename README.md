# 📋 教案评估 (Vocational Lesson Audit)

> AI 逆向教研员 — 挑刺、评估、重构职业学校教案

## 概述

面向中职/高职教师的教案审计 Skill。AI 扮演"逆向教研员"角色，对教案进行六维深度审计，确保教学设计符合**岗课赛证融通**和**行动导向**的职教要求。

## ✨ 核心特色

| 特色 | 说明 |
|------|------|
| **逆向挑刺** | 拒绝讨好型回复，直指教案硬伤 |
| **6维审计矩阵** | 守门检查 → 目标 → 问题链 → 活动 → 辅助 → 学情 |
| **百进制评分** | 100分制4级评定（优秀/良好/合格/待改进） |
| **课程类型识别** | 专业课自动启用工作过程属性检查 |
| **渐进式披露** | 分层引用参考资料，避免一次性信息轰炸 |

## 📐 六维审计矩阵

| 维度 | 满分 | 检查要点 |
|------|------|---------|
| 守门检查 | 15分 | 课堂类型、课题规范、重难点措施 |
| 目标审计 | 20分 | 核心问题、可测性、岗课对接、思政融入 |
| 问题链诊断 | 15分 | 伪互动排查、思维递进、行动导向六步法 |
| 活动反假 | 20分 | 方法选择、真假活动、评价闭环、考核标准 |
| 辅助审计 | 20分 | 时间分配、板书设计、语言规范、反思框架 |
| 学情审计 | 10分 | 已有基础、预判困难、差异应对 |

## 🚀 安装

1. 将此目录复制或链接至 OpenCode 技能目录：
   ```bash
   cp -r 教案评估 ~/.config/opencode/skills/lesson-plan-eval/
   ```
2. 确保 `SKILL.md` 为该目录下的主文件

## 💡 使用方法

**触发词**：`教案评估`、`教案检查`、`教学设计改进`、`教案评分`

**输入方式**：
- 直接粘贴教案文本
- 上传 `.docx` / `.pdf` / `.md` 文件

**执行流程**：
1. 检测完整性 → 缺失时输出补全指南
2. 识别课程类型 → 专业课启用六步法检查
3. 六维审计 → 生成评分与硬伤表
4. 输出报告 → 询问是否需要导出 `.docx`/`.md`

## 🏷️ 评分标准

| 分值 | 结论 | 行动建议 |
|------|------|---------|
| 90-100分 | 优秀 → 可直接实施 | 建议分享推广 |
| 75-89分 | 良好 → 稍作调整 | 按硬伤表逐条修正 |
| 60-74分 | 合格 → 需重点修改 | 优先修正核心硬伤 |
| <60分 | 待改进 → 需重新设计 | 建议教研组研讨 |

## 📚 依赖标准

- 布鲁姆教育目标分类法
- 5E 教学模式（Engage → Explore → Explain → Elaborate → Evaluate）
- 行动导向六步法（资讯 → 计划 → 决策 → 实施 → 检查 → 评估）
- 岗课赛证融通（岗位对接 + 课堂 + 竞赛 + 证书）
- 课程思政自然融入
