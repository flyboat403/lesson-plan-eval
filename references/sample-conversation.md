# 评估对话示例

> 本文档展示典型教案评估的对话流程，帮助理解skill的实际使用方式。

---

## 示例1：完整教案评估

**用户**: "帮我评估这份心理健康教案：[粘贴教案全文]"

**AI响应流程**:
```
[加载skill lesson-plan-eval]
→ 课程类型识别：公共基础课（心理健康）
→ 守门检查：内容完整，进入审计
→ 六维度审计：
  - 维度A：目标审计（核心问题/可测性/思政融入）
  - 维度B：问题链诊断（伪互动排查/思维递进）
  - 维度C：活动反假（方法选择/活动必要性）
  - 维度D：辅助审计（时间/板书/语言）
  - 维度E：学情审计（已有基础/预判困难）
→ 计算得分：68/100分 → 合格
→ 输出审计报告
→ 询问："是否需要生成详细报告文档？"
```

## 示例2：教案补充后评估

**用户**: "我教数控机床维修"

**AI响应**:
```
[守门检查失败]
→ 输出《教案补全指南》：
"老师您好，教案内容不足，请补充：
①教学目标（知识/技能/素养侧重）
②重难点（学生最易卡点）
③教学过程（大致环节）"
```

**用户补充**: "目标是让学生掌握数控故障排查，重点是报警代码解读..."

**AI响应**:
```
[自动触发重新评估]
→ 课程类型识别：专业课（含"数控""故障""操作"）
→ 守门检查：内容完整
→ 六维度审计（启用专业课专项检查：工作过程标注）
→ 输出完整审计报告
```

---

## 示例3：专业课专项检查

**用户**: "评估这份汽车维修实训教案"

**AI响应特点**:
```
[课程类型识别：专业课]
→ 启用维度B额外检查：
  - 各环节是否标注工作过程属性？
  - 行动导向六步法是否完整？
→ 输出课堂结构表含"工作属性"列：
  | 环节 | 教师活动 | 学生活动 | 意图 | 时间 | 工作属性 |
  | 导入 | 播放故障视频 | 观察记录 | 激发兴趣 | 5min | 资讯 |
```

## 示例5：边缘触发场景

**用户**: "帮我看下这份教案有啥问题"（未明确说"评估"）

**AI响应**:
```
[Description触发：含"教案"+"问题"关键词]
→ 自动加载lesson-plan-eval
→ 执行评估而非简单回答
→ 输出挑刺式审计报告
```

**用户**: "这份教案怎么样"

**AI响应**:
```
[Description触发：含"教案"+"怎么样"]
→ 不简单回答"挺好"或"还行"
→ 执行逆向审计，指出具体问题
```

## 对话注意事项

| 用户表述 | 应触发skill | 不应触发 |
|---------|------------|---------|
| "帮我评估教案" | ✓ | — |
| "教案有问题吗" | ✓ | — |
| "这份教案怎么样" | ✓ | — |
| "帮我看教案" | ✓ | — |
| "写一份教案" | — | ✓（创作而非评估） |
| "教案模板" | — | ✓（索取而非评估） |
| "什么是教案" | — | ✓（概念而非评估） |
