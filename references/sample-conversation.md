# 评估对话示例

> 本文档展示典型教案评估的对话流程，帮助理解skill的实际使用方式。

---

## 示例1：公共基础课完整评估（语文课）

**用户**: "帮我评估这份语文教案：《荷塘月色》精读课"

**AI响应流程**:
```
[加载skill lesson-plan-eval v0.4.0]
→ 课程类型识别：公共基础课（语文）
→ 守门检查：课型/课题/重难点措施完整，进入审计
→ 八维度审计：
   A 基本信息审计(8/10): 课题规范但缺少课题序号，授课班级信息不完整
   B 学情审计(9/12): 有基础分析（已学过记叙文阅读方法），但预判困难笼统（"学生可能难以理解"），缺少元认知策略设计
   C 教学目标审计(10/15): 使用模糊词"提升感知""体会情感"，重难点无具体措施支撑
   D 教法审计(8/10): 学习结果识别为言语信息+态度，教法选择为朗读法+讨论法，基本匹配但深度不够
   E 活动审计(12/15): 小组合作讨论有效，但考核评价标准不明确，缺少过程性评价设计
   F 问题链审计(10/12): 追问设计有递进（"写了什么→为什么这样写→换成你會怎样写"），但可深化至文本细读层面
   G 辅助审计(7/12): 时间分配合理（导入5min/精读20min/讨论15min/总结5min），板书设计完整，但教学反思空白
   H 教学评一致性审计(7/14): 目标中"体会情感"与评价方式（简答题）不匹配，布鲁姆层级存在错位（目标为分析级，评价仅达记忆级）
→ 计算得分：71/100分 → 合格
→ 输出含教法匹配诊断表的审计报告
```

**审计报告片段**（教法匹配诊断表）:
| 教学环节 | 学习结果类型(加涅) | 选用教法 | 匹配度 | 诊断建议 |
|---------|------------------|---------|-------|---------|
| 导入环节 | 言语信息 | 讲授法 | ★★★★ | 匹配，但可增加情境激发 |
| 精读环节 | 智慧技能 | 朗读法+讨论法 | ★★★ | 基本匹配，建议增加思维可视化工具 |
| 讨论环节 | 态度 | 小组合作 | ★★★ | 缺少具体评价标准 |

## 示例2：教案补充后评估（数控机床维修专业课）

**用户**: "我教数控机床维修"

**AI响应**:
```
[守门检查失败]
→ 输出《教案补全指南》v0.4.0：
"老师您好，教案内容不足，请补充：
①基本信息（课程名称、授课班级、课时）
②教学目标（知识/技能/素养侧重，需明确学习结果类型）
③重难点（学生最易卡点及突破措施）
④教学过程（大致环节，需标注工作过程属性）"
```

**用户补充**: "目标是让学生掌握数控故障排查，重点是报警代码解读..."

**AI响应**:
```
[自动触发重新评估 - v0.4.0]
→ 课程类型识别：专业课（关键词：数控、故障、操作、维修）
→ 守门检查：内容完整，进入审计
→ 八维度审计（启用专业课专项检查）：
   - 工作过程属性检查（资讯→计划→决策→实施→检查→评估）
   - 岗位能力对接检查（工业机器人操作员/数控设备装调工）
→ 输出完整审计报告，含课堂结构表（含"工作属性"列）
```

## 示例3：专业课专项检查（汽修实训课）

**用户**: "评估这份汽车维修实训教案"

**AI响应特点**:
```
[加载skill lesson-plan-eval v0.4.0]
[课程类型识别：专业课 - 汽修实训]
→ 启用专业课专项检查：
   1. 加涅学习结果分类识别
   2. 工作过程属性标注
   3. 教法匹配度诊断
→ 八维度审计：
   D 教法审计：识别学习结果为"智慧技能（规则运用）"，教法选用"演示法+实操法"，与工作过程导向匹配
   E 活动审计：检查行动导向六步法完整性（资讯→计划→决策→实施→检查→评估）
→ 输出加涅分类标注表：
   | 教学环节 | 加涅学习结果类型 | 工作属性 | 教法建议 |
   |---------|----------------|---------|---------|
   | 故障呈现 | 言语信息 | 资讯 | 讲授法 |
   | 方案制定 | 智慧技能 | 计划+决策 | 头脑风暴 |
   | 实施维修 | 动作技能 | 实施 | 演示法+实操法 |
   | 成果检查 | 智慧技能 | 检查 | 互评+师评 |
```

## 示例5：边缘触发场景

**用户**: "帮我看下这份教案有啥问题"（未明确说"评估"）

**AI响应**:
```
[Description触发：含"教案"+"问题"关键词]
→ 自动加载lesson-plan-eval v0.4.0
→ 执行评估而非简单回答
→ 输出挑刺式审计报告（含八维度硬伤表）
```

**用户**: "这份教案怎么样"

**AI响应**:
```
[Description触发：含"教案"+"怎么样"]
→ 不简单回答"挺好"或"还行"
→ 执行逆向审计
→ 指出具体问题：
   - C教学目标：模糊词"提升/感知"，无法测量
   - H一致性：目标为分析级，评价为记忆级
   - E活动：小组讨论缺少考核标准
→ 输出评分：XX/100分 → 结论
```

## 对话注意事项

| 用户表述 | 应触发skill | 不应触发 | 备注 |
|---------|------------|---------|------|
| "帮我评估教案" | ✓ | — | 直接加载v0.4.0执行八维度审计 |
| "教案有问题吗" | ✓ | — | 触发审计，输出硬伤表 |
| "这份教案怎么样" | ✓ | — | 执行逆向审计，不简单敷衍 |
| "帮我看教案" | ✓ | — | 同上 |
| "教案评分" | ✓ | — | 输出百分制评分 |
| "写一份教案" | — | ✓ | 创作模式，非评估 |
| "教案模板" | — | ✓ | 索取引用，非评估 |
| "什么是教案" | — | ✓ | 概念解释，非评估 |
| "帮我改成行动导向" | — | ✓ | 重构请求，非评估 |
